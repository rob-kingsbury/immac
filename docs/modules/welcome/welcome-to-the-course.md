# Welcome to the Course

Welcome to Web Components. By the end of this semester you'll have built a multi-page website from scratch, using the same structural language every website on the internet is built from: HTML. This first week is deliberately light. No coding yet. Just a look at what this course covers, what HTML actually is, and what software to have ready before next class.

## What this course covers

This course is about structure and meaning, not appearance. You'll learn semantic HTML: the elements that describe what a piece of content *is* (a heading, a list, a navigation menu, an article) rather than how it looks. Styling those elements, colours, layout, spacing, is the subject of your Web Styles course, taught right after this one each week. The two courses build the same project from two different angles: this one gives it structure, the other gives it a visual design.

Over the semester you'll cover, in order: the building blocks of a web page, how to organize a page and a whole site, images and media, accessibility, forms, search engine visibility, and code quality. In the last two weeks you'll bring everything together into a finished, multi-page project.

Every piece of work you submit in this course is a real, live website, hosted on GitHub Pages under your own account. Next week is entirely dedicated to setting that up.

## What is HTML?

HTML stands for **HyperText Markup Language**. Every website you have ever visited, from the simplest personal blog to the largest search engine, is built on it. Understanding what it does, and just as importantly what it *doesn't* do, is the first real concept in this course.

**HTML describes structure and meaning, not appearance.** When you write HTML, you're labelling content: this is a heading, this is a paragraph, this is a list, this is a link. You are not saying what colour it is, how large the text should be, or where it sits on the page. That's a separate job, done by CSS, which you'll learn in your Web Styles course. Keeping those two jobs separate, structure in HTML, appearance in CSS, is one of the most important habits this course will build in you.

**A browser reads HTML and turns it into the page you see.** Your browser (Chrome, Firefox, Edge, Safari) is a program that requests a file, receives HTML back, and renders it into the visual page in front of you. Without any styling at all, an HTML page still displays, just plainly: headings look bigger than paragraphs, lists show bullets, links are blue and underlined. That plain, unstyled rendering is the browser's built-in idea of what each HTML element means, before any designer touches it.

**"Markup" means the tags are mixed in with the content they describe.** A piece of HTML looks like `<h1>Welcome</h1>`, the word "Welcome" marked up as a top-level heading by the tags surrounding it. You'll write your first real tags in week three. For now, the concept to hold onto is this: HTML is a language for labelling what things *are*.

## How a website reaches your screen, briefly

You don't need the full technical picture yet, but a rough mental model helps everything that follows make sense. Every website involves two computers: a **client** (your browser, on your device) and a **server** (another computer, somewhere else, storing the website's files). When you visit a page, your browser asks the server for it, the server sends the HTML file back, and your browser displays it. That's the entire relationship this course, and the modern web, is built on.

## Software to have ready for next class

Next week is entirely hands-on setup, installing and configuring everything you'll use all semester. Having these downloaded *before* that class saves real time, since installers can take a while and some require a restart.

- **[Visual Studio Code](https://code.visualstudio.com/)**, the code editor you'll write every line of HTML in this semester. Free, and available for Windows, macOS, and Linux.
- **[Git](https://git-scm.com/downloads)**, the version control tool that tracks changes to your code and connects it to GitHub. Download the installer for your operating system, but don't worry about configuring anything yet. That's next week's class.
- **A GitHub account.** If you don't already have one, create a free account at [github.com](https://github.com/). Use an email address you'll still have access to after you graduate.
- **A modern browser** with developer tools, Chrome, Firefox, or Edge all work well. If you already use one daily, you're set.

Download these, but leave them uninstalled or unconfigured if you'd rather do that step-by-step in class next week. Either is fine.
